# Support

## Usage and development questions

Read the [README](README.md), command help, and
[DEVELOPMENT.md](DEVELOPMENT.md) first. If the answer is not covered, open a
GitHub issue with a minimal reproduction, the heyfood version, Python version,
operating system, expected behavior, and redacted output.

Public issues are appropriate for reproducible CLI bugs, documentation gaps,
and feature proposals. They are not appropriate for account recovery, billing,
security reports, production service incidents, or personal dietary guidance.

For account or hosted-service help, contact `hi@frntr.ai`. Follow
[SECURITY.md](SECURITY.md) for vulnerabilities.

Never post access tokens, API keys, config files, private locations, meal
history, or health and dietary profile data. Replace real values with synthetic
examples before sharing diagnostics.
